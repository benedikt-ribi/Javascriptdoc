cookie Cookies sind Strings, die mit dem Browser auf dem Gerät des Users gespeichert werden. • Cookies sind über document.cookie zugänglich • Cookies können ein Ablaufdatum haben. • User/Browser können das Ablaufdatum übersteuern und Cookie vorzeitig löschen. • User können das Setzen von Cookies verhindern. • Die Cookies für eine Domain sind in einem String, durch Semikola getrennt. Web Storage

• Mit HTML5 wurde clientseitiges Speichern, sogenanntes Web Storage, eingeführt. • Heute ist Webstorage eine eigene API. • Web Storage stellt zwei Objekte zur Verfügung: • window.localStorage für unbegrenzte Speicherung. • window.sessionStorage zum Speichern von Daten für eine Session. localStorage und sessionStorage sind, bis auf das Verfallsdatum, identisch. • localStorage und sessionStorage sind nicht nur von der Domain sondern auch vom Protokoll abhängig. Daten, die in http erstellt werden stehen in https nicht zur verfügung. • Die Speicherkapazität ist Browserabhängig Test (min 5MB): http://dev-test.nemikor.com/web-storage/support-test/ • Der User kann den Web Storage jederzeit löschen.
